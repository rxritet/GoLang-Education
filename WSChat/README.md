# 10 — WSChat — WebSocket Chat

Чат в реальном времени: сервер получает сообщения от одного клиента и мгновенно рассылает их всем остальным (Broadcast Hub).

## Применено

- Пакет `github.com/gorilla/websocket` для апгрейда и read/write буферов.
- Паттерн Hub: центральная структура, управляющая мапой подключений `map[ClientInterface]bool` в одной горутине.
- Изоляция конкурентного доступа через каналы `Register`, `Unregister`, `Broadcast`. Мьютексы не используются.
- Две дочерние горутины на каждого клиента (Gorilla rule): `ReadPump` (слушает сокет и пишет в hub) и `WritePump` (читает из hub и пишет в сокет).
- Ping/Pong keepalive mechanism.
- Адаптивный веб-интерфейс чата с тёмной темой.

## Структура

```
WSChat/
├── main.go            # точка входа, CLI / interactive, HTML UI, HTTP+WS роутинг
├── hub/
│   └── hub.go         # центральный broadcast hub (избегает циклической зависимости)
└── client/
    └── client.go      # реализация WebSocket client (readPump, writePump)
```

## Быстрый старт

```bash
# Интерактивный режим
cd WSChat && go run .

# Или с флагами
go run . --port 8080
```

После запуска:
1. Открой `http://localhost:8080` в двух+ разных браузерах/вкладках.
2. Введи разные юзернеймы в модальном окне.
3. Отправь сообщение — оно мгновенно появится во всех вкладках.

## Сообщения

Сообщения циркулируют между горутинами в формате JSON `[]byte`:

### Внутренний формат рассылки:
```json
{
  "user": "Alice",
  "text": "Hello, world!"
}
```

Фронтенд имеет стилизацию для сообщений от других ("row-other"), от себя ("row-self") и системных событий ("row-system", например "Alice joined the chat").
